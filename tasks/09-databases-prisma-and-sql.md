# 09 - Databases Prisma And SQL

## Goal

Learn the database layer well enough to build real full-stack apps.

## Learn

- relational thinking
- PostgreSQL basics
- tables and relationships
- primary and foreign keys
- normalization basics
- queries
- Prisma models and migrations
- backend data access patterns

## Tasks

- choose PostgreSQL as the default learning database
- model users, projects, tasks, comments, and sessions
- write down the relationships before coding them
- add Prisma to your backend
- create migrations
- fetch related data cleanly
- replace mock data with real database data

## Mini exercises

- create one-to-many project and task data
- fetch a project with all tasks
- create a comment linked to a task and user

## Done when

- you understand your data model before writing endpoints
- database work feels practical instead of abstract
- you can persist the main app features with confidence
