# 07 - Data Fetching And Frontend Architecture

## Goal

Learn how to structure frontend applications that consume APIs cleanly.

## Learn

- fetch and async flow
- API layers
- loading, error, and empty states
- optimistic thinking
- separation of concerns
- feature-based folders
- basic caching concepts
- optional React Query next

## Tasks

- create a feature folder structure
- create API functions outside page components
- fetch projects and tasks from an API
- transform raw API data before rendering it
- handle pending, error, empty, and success states
- refactor large page files into smaller pieces
- compare manual fetch logic with React Query later

## Mini exercises

- create a `getProjects` function
- create a reusable loading component
- create an error fallback component
- map backend data into UI card data

## Done when

- your pages are not full of fetch code
- your data flow is easier to reason about
- you can structure a scalable React app
