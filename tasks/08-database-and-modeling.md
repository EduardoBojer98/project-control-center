# 08 - Database And Data Modeling

## Goal

Learn how app data is stored and related.

## Learn

- database basics
- tables or collections
- ids
- relationships
- one-to-many relationships
- migrations
- ORM basics such as Prisma
- SQL basics if using PostgreSQL

## Tasks

- choose a database: PostgreSQL is a strong default
- design tables for users, projects, tasks, comments
- create the schema
- connect your backend to the database
- save and read project data from the database
- stop using mock data for that feature

## Mini exercises

- create a task that belongs to a project
- create a comment that belongs to a task
- fetch one project with its tasks

## Done when

- you understand how your data is modeled
- you can move from mock data to persisted data
- you can explain relationships between tables
